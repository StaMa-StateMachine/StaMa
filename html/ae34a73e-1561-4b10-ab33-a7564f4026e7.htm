Integrating the State Machine with the Application

Explains topics to be considered when embedding a StaMa based state machines into an application.

StaMa based state machines do not require or enforce a specific application infrastructure beyond a few basic .NET Windows Desktop Framework or .NET Micro Framework functionalities. If is assumed that the StaMa State Machine Controller Library can be ported to other .NET Framwork derivatives without major effort.

The following sections provide a few additional topics to consider in order to get the most out of the library:

  • Error and Exception Handling

  • Tracing State Changes, Transition Testing and Event Dispatching
